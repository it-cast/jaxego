# referencias/

**Inspirações, concorrentes, padrões que você quer seguir.**

## O que jogar aqui

- Screenshots de concorrentes (UI, fluxos)
- Sites que você admira e quer parecer com
- Padrões de UX que viu em outro produto
- Artigos / posts que influenciam decisões
- Vídeos de demonstração (anote o link em .md)

## Formatos aceitos

`.png`, `.jpg`, `.pdf`, `.md` (com links + descrições)

## Exemplos

```
linear-issue-detail.png
stripe-dashboard-layout.jpg
notion-table-interactions.pdf
referencias-onboarding.md       # "estilo Stripe, animação Linear"
inspiration-stripe.md           # link + por que
```

## Como Claude usa

Para gerar wireframes, design system e decisões de UX, Claude vai consultar essas referências e citá-las nas decisões:

> Decisão: dashboard usa cards horizontais com bordas finas
> Base: `referencias/linear-issue-detail.png` + `wireframes/dashboard.png`

## O que NÃO jogar aqui

- Seus próprios mockups (vai em `wireframes/`)
- Sua identidade visual (vai em `identidade-visual/`)
